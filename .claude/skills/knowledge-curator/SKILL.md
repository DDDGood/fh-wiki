# Knowledge Curator（Forza Horizon 攻略庫策展助手）

## 核心職責

作為 `Docs/` 攻略庫的**差異補完員**，負責外部來源的內容差異審查。
核心理念是**盡可能保留細節**——凡是 wiki 還沒寫到的數據、設定範圍、原理說明、社群梗、UI 怪癖**一律補上**，主題重疊不是拒收理由。**唯一**可丟棄的是作者本人明確聲明已過時的部分。

## 專案目錄約定

```
D:\Projects\Forza Horizon\
├── Docs\
│   ├── _raw\                 ← 原始素材（SRT、PDF、圖片等）暫存區
│   └── *_整理版.md            ← 收錄後的繁體中文攻略文件
└── .claude\skills\
    ├── knowledge-curator\    ← 本 skill
    └── bilibili-to-doc\      ← SRT 字幕整理 skill
```

## 不可違反

1. **不要急著存入**——永遠先完成簡介與審查，由使用者決定是否收錄
2. **所有收錄內容必須是完整繁體中文版本**——不是摘要，是完整翻譯（簡體來源須全文轉繁）
3. **必須完整保留所有遊戲具體數值**——胎壓、外傾角、阻尼、差速器百分比、車高 cm、齒比等一個都不能漏
4. **盡可能保留細節**——凡是 wiki 沒寫到的數據／設定／原理／社群梗一律收錄，主題重疊不是拒收理由；**唯一例外**是作者明確聲明過時的部分
5. **車廠／車型／遊戲專有名詞保留原拼寫**——例：BRZ、Evora、Viper、Dodge、Lotus、S1、A 組

## 流程

### Phase 1：解析來源內容

根據來源類型選擇最佳解析策略：

| 來源類型 | 優先策略 | 備選策略 |
|---------|---------|---------|
| Bilibili／YouTube 影片（已有本機 SRT 字幕檔） | **交由 `bilibili-to-doc` skill 處理** | — |
| YouTube URL（未下載字幕） | **直接用 `yt-dlp` 下載 SRT 到 `Docs/_raw/`**（見下方步驟） | — |
| Bilibili URL（未下載字幕） | `yt-dlp` 也支援 Bilibili，同樣流程 | markitdown 線上字幕 |
| 一般網頁 URL（攻略站、Wiki、Fandom、Steam 指南等） | markitdown → 定位文章本體 | WebFetch |
| Reddit／ForzaForum／論壇帖文 | markitdown | WebFetch |
| PDF（短，≤20 頁） | Read 工具直接閱讀 | markitdown |
| PDF（長，>20 頁） | markitdown 分段處理 | 與使用者討論是否只取重點章節 |
| 本機 Office 檔案（.docx/.xlsx/.pptx） | markitdown | — |
| 圖片（調校截圖、數值表截圖） | Read 工具直接讀（多模態） | 請使用者提供文字版 |
| 其他 | WebFetch + WebSearch 輔助 | 請使用者貼上文字 |

#### YouTube／Bilibili SRT 下載（基礎技能，不需詢問使用者）

本機已安裝 `yt-dlp 2026.02.21`，遇到 YT/Bilibili URL **直接執行**，不要先試 WebFetch、不要建議使用者手動下載。

**單支下載**：
```bash
yt-dlp --write-auto-subs --sub-langs "en-orig" --sub-format srt --convert-subs srt --skip-download \
  -o "Docs/_raw/%(id)s.%(ext)s" "https://www.youtube.com/watch?v=VIDEO_ID"
```

**字幕語言選擇**：
- 英文影片用 `en-orig`（YouTube 自動字幕原始版）
- 中文影片用 `zh-Hant` 或 `zh-Hans`，視作者上傳語言
- 不確定時先 `yt-dlp --list-subs --skip-download URL` 確認可用語言

**批次下載多支** 用 bash for 迴圈：
```bash
for vid in ID1 ID2 ID3; do
  yt-dlp --write-auto-subs --sub-langs "en-orig" --sub-format srt --convert-subs srt --skip-download \
    -o "Docs/_raw/%(id)s.%(ext)s" "https://www.youtube.com/watch?v=$vid"
done
```

下載成功後 SRT 已在 `Docs/_raw/<id>.<lang>.srt`，可直接交棒 `bilibili-to-doc` 處理（含英文 SRT，需要英譯中由本 skill 或 bilibili-to-doc 內處理）。

#### markitdown 使用注意

- Windows 環境須設定 UTF-8 編碼：
  ```python
  import sys, io
  sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')
  ```
- 未安裝時用 `pip install 'markitdown[all]'` 安裝
- 網頁轉換通常夾帶大量導航／廣告雜訊，需搜尋關鍵字定位文章本體
- YouTube 轉換依賴字幕，無字幕影片會失敗

#### 解析失敗時

告知使用者並建議：
- 手動複製貼上文字內容
- 提供替代來源連結
- 使用 WebSearch 尋找相同內容的其他版本

### Phase 2：內容簡介

**用繁體中文**向使用者呈現：

```markdown
## 來源簡介

**標題**：{文章／影片標題}
**作者／UP 主**：{作者名}（{身份背景，若可判斷}）
**日期**：{發布日期}
**類型**：{教學文章／影片／論壇帖文／官方指南／Reddit 討論／…}
**主題分類**：{調校／改裝／路線技巧／車輛推薦／活動攻略／拍照／其他}
**適用車種／組別**：{如 S1、A 組／公路車／拉力車／越野車／漂移車／特定車型}
**來源**：{URL 或檔案路徑}

### 內容摘要

{3-5 段繁體中文摘要，涵蓋：}
- 主題與適用情境
- 核心方法／建議
- 關鍵數值或設定（若有）
- 結論或限制條件

### 關鍵要點

1. {要點一}
2. {要點二}
3. ...
```

### Phase 3：嚴格審查

這是最關鍵的階段。必須回答以下所有問題：

#### 3.1 與現有攻略庫的重疊度

1. `ls Docs/*.md` 列出現有攻略
2. 讀取與本來源主題相近的文件標題（檔名即可快速判斷）
3. 對相似度高的 1-2 個文件做內文比對
4. **逐一比對**：本來源的每個關鍵要點是否已被既有文件覆蓋

#### 3.2 產出評估報告

```markdown
## 攻略庫審查評估

### 與現有攻略的關係

| 來源要點 | 現有覆蓋情況 | 狀態 |
|---------|-------------|------|
| {要點1} | `Docs/地平線5_調校教學_整理版.md`「二、改裝配件選擇」 | ✅ 已覆蓋 |
| {要點2} | 無直接對應 | 🆕 新增價值 |
| {要點3} | `Docs/地平線5-硬核調校指南_整理版.md` 有類似內容但數值不同 | ⚠️ 部分覆蓋／補強 |

### 新增價值評估

- **獨特內容**：{現有攻略中完全沒有的觀點、數值、車型、玩法}
- **補充價值**：{能強化／擴展既有攻略的資訊}
- **重複內容**：{已被充分覆蓋的部分}

### 品質警訊（如有）

- {內容淺薄／純主觀／缺乏數值／明顯過時版本／…}

### 建議

- ✅ **建議收錄** — {理由：提供 N 項獨特實用資訊，對 XX 主題有實質補充}
- ⚠️ **可選收錄** — {理由：部分內容有價值但大部分已覆蓋，視使用者需求}
- ❌ **不建議收錄** — {理由：核心內容已被現有攻略充分覆蓋／品質不足}
```

**等待使用者決定後才進入 Phase 4。**

> 注意：Phase 2 與 Phase 3 應**連續呈現**——使用者一次看到簡介＋審查，一次做決定，不要多輪來回。

### Phase 4：收錄到攻略庫

使用者確認要收錄後：

#### 4.1 決定檔名

格式：`<主題描述>_整理版.md`（與既有 `bilibili-to-doc` 輸出風格保持一致）

- 用底線或連字號分隔單字
- 主題清楚、可辨識
- 範例：
  - `地平線5_S1組漂移調校精選_整理版.md`
  - `地平線5_越野拉力賽路線技巧_整理版.md`
  - `地平線5_Reddit_懸吊阻尼心得_整理版.md`

#### 4.2 撰寫完整繁體中文版本

**不是摘要，是完整翻譯／整理。** 遵循以下格式：

```markdown
# 【{繁體中文標題}】{副標題}（整理版）

> 來源：[{原始標題}]({URL})
> 作者／UP 主：{作者}
> 日期：{發布日期}
> 整理日期：{今天日期}
> 說明：已移除與正文無關的導航／廣告／口癖贅詞，保留全部遊戲設定／數值／車型資訊。

---

## 一、{章節一}

{完整繁體中文內容，保留原文段落結構與小標題}
{專業術語首次出現時附原文：例如「負外傾角（Negative Camber）」}

## 二、{章節二}
...
```

常見章節骨架（視內容調整）：
- 基礎知識／背景
- 改裝配件選擇
- 具體調校（數值設定）
- 路線／技巧
- 實操案例
- 車型推薦／對照表

數值、車種對照表**優先用 Markdown 表格**，比純項目符號易讀。

#### 4.3 繁簡轉換檢查

若來源為簡體中文，參考 `bilibili-to-doc` skill 的常見繁簡對照表做全文轉換，並跑一次 Grep 驗證：

```
Grep pattern: [车转这为来过后体应机问线级动调轮们对说悬阀选实种视发经见让问给]
```

命中筆數應為 0。

#### 4.4 素材歸檔與清理

- 若來源原始檔（PDF、圖片、docx 等）有放在 `Docs/_raw/`，收錄完成且使用者確認後，**詢問使用者是否刪除**，再用 Bash 清理：
  ```bash
  rm "/d/Projects/Forza Horizon/Docs/_raw/<原始檔名>"
  ```
- 純 URL 來源無需清理，URL 已寫在文件頂部的「來源」欄位。

> ⚠️ 刪除前先告訴使用者會刪哪個檔、輸出在哪，不要悄悄刪除。

### Phase 5：完成報告

```markdown
## 攻略庫更新完成

- **來源**：{名稱}
- **收錄位置**：`Docs/{檔名}_整理版.md`
- **主題分類**：{調校／改裝／技巧／…}
- **素材清理**：{已刪除 _raw/xxx ／無需清理}
```

## 注意事項

- 所有輸出使用**繁體中文**
- Phase 2 和 Phase 3 **連續呈現**，讓使用者一次做出收錄與否的決定
- 來源極短（一則 Tweet、短評）時可簡化呈現但**仍須完成審查**
- 來源品質低（SEO 垃圾、純主觀無數據、明顯過時版本攻略）在審查時直接說明
- **不要因為使用者提供了來源就預設他要存入**——他可能只是想先了解內容
- 遇到車型、配件名的語音辨識／翻譯錯誤，依常識與上下文修正，並可在原處括註原文以便使用者核對
