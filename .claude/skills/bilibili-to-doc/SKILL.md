# bilibili-to-doc

把 Bilibili 下載下來的 SRT 字幕（通常是 Forza Horizon 等遊戲教學影片）整理成乾淨、結構化、**繁體中文**的 Markdown 技術文件。

## 專案目錄約定

```
D:\Projects\Forza Horizon\
├── Docs\
│   ├── _raw\           ← 放入原始 .srt 檔案（處理完成後刪除）
│   └── *.md            ← 輸出的整理版 Markdown
└── .claude\skills\bilibili-to-doc\SKILL.md
```

- **輸入**：`Docs\_raw\*.srt`
- **輸出**：`Docs\<影片主題>_整理版.md`（繁體中文）
- 轉換完成並確認檔案無誤後，**刪除 `_raw/` 裡對應的 SRT**。

## 執行流程

當使用者請求處理 `_raw/` 裡的字幕（或指定某個 SRT 檔）時，依序完成：

### 1. 讀取與分塊

- 以 `Read` 工具讀取 SRT。檔案通常很大，先用 `offset` / `limit` 分段讀取（每段 800 行左右）。
- 完整讀完再開始整理，避免漏掉結尾資訊。

### 2. 清洗雜訊

從原始字幕中 **移除**：

- SRT 的序號行（`1`, `2`, `3`...）
- 時間軸（`00:01:23,456 --> 00:01:25,789`）
- 空行
- **BGM 歌詞**：整段連續的英文歌詞（通常伴隨 `♪` 符號或出現 `bgm` 字樣），以及明顯的音樂哼唱區段。常見特徵：
  - 含 `♪` 前後綴
  - 連續多行英文且語意不連貫（例如 `you take one / i take one / can't hide you`）
  - 內含 `bgm`、`payback here`、`look at the sky` 之類反覆出現的歌詞片段
- **UP 主個人贅詞**：自我宣傳（「三連」「點讚」「感冒」「嗓子」）、口癖（「各位觀眾老爺」「好了」「那我們」「呃」）、閒聊片段。
- 明顯的語音辨識錯誤音譯，例如把 UP 主自稱「up」辨識為「app」，應根據上下文還原成「作者／UP 主」或刪除。

### 3. 保留所有遊戲資訊

**必須完整保留**：

- 所有具體的調校數值（胎壓、外傾角、阻尼、差速器百分比、車身高度 cm 等）
- 車輛型號、引擎型號、配件名稱
- 改裝邏輯與原理說明
- 各車種（公路車／拉力車／越野車／漂移車）的分別建議
- 圖表座標、紅黃線、橫縱軸等遊戲 UI 說明
- 按鍵操作（例如 `按 T 打開遙測`、`按 Y 查看數據`）

### 4. 繁體中文轉換

**整份輸出必須是繁體中文**。常見需替換的簡體字：

| 簡 | 繁 | 簡 | 繁 | 簡 | 繁 |
|---|---|---|---|---|---|
| 车 | 車 | 转 | 轉 | 轮 | 輪 |
| 动 | 動 | 调 | 調 | 这 | 這 |
| 们 | 們 | 对 | 對 | 为 | 為 |
| 过 | 過 | 来 | 來 | 后 | 後 |
| 体 | 體 | 应 | 應 | 说 | 說 |
| 悬 | 懸 | 阀 | 閥 | 并 | 並 |
| 选 | 選 | 级 | 級 | 实 | 實 |
| 种 | 種 | 线 | 線 | 样 | 樣 |
| 视 | 視 | 发 | 發 | 机 | 機 |
| 经 | 經 | 问 | 問 | 给 | 給 |
| 见 | 見 | 让 | 讓 | 实际 | 實際 |

**寫完後務必用 `Grep` 掃一次剩餘簡體字**（見步驟 6 的驗證清單），有漏網的就用 `Edit` 補修。

注意：遊戲／車廠專有名詞保留原拼寫（例：BRZ、Evora、Viper、Dodge、Lotus、BGM）。

### 5. 文件結構

輸出的 Markdown 通常依此骨架組織（視影片內容調整章節）：

```markdown
# 【<影片標題>】<副標>（整理版）

> 來源：Bilibili <BV 號>
> 說明：已移除時間軸、BGM 字幕與口癖贅詞，保留全部遊戲改裝／調校資訊。

---

## 一、基礎知識
## 二、改裝配件選擇
## 三、具體調校（數值設定）
## 四、實操案例
## 五、上傳／分享調校
```

子章節使用 `###`，數值、車種對照表用項目符號與 `**粗體**` 標示重點。長列表使用 Markdown 表格。

### 6. 驗證清單

在輸出前確認：

- [ ] 沒有任何時間軸（搜尋 `-->` 應為 0 筆）
- [ ] 沒有 BGM 歌詞區段（搜尋 `♪` 應為 0 筆）
- [ ] 沒有孤立數字行（SRT 序號殘留）
- [ ] 全繁體（跑一次 Grep：`[车转这为来过后体应机问线级]` 應為 0）
- [ ] 保留所有數值（回頭對照原檔確認重要數字未漏）
- [ ] 檔名格式：`<主題>_整理版.md`

### 7. 清理 _raw

輸出檔案寫入 `Docs/` 並經使用者確認後，用 `Bash` 刪除對應 SRT：

```bash
rm "/d/Projects/Forza Horizon/Docs/_raw/<原始檔名>.srt"
```

> ⚠️ **刪除前先告訴使用者**將刪哪個檔、輸出在哪，等待回應或在同一輪訊息中一併處理，不要悄悄刪除。

## 其他提示

- 使用者可能一次丟多個 SRT 到 `_raw/`。先 `ls` 列出，逐個處理。
- 不同 UP 主風格不同，有些影片沒有 BGM 段落、有些則整段唱歌；清洗策略保持一致但結構可以彈性。
- 若遇到語音辨識明顯錯誤（例如車型、配件名），以常識與上下文修正，並在文件中以括號標註原文或保留 `（原音近似）` 提醒使用者核對。
- 對照表、數值組合優先用 Markdown 表格呈現，比純項目符號更易讀。
